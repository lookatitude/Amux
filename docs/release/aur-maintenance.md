# AUR maintenance — `amux-bin` (D6)

T3 devops · work package D6. Procedure for updating the Arch User Repository
**binary** package after a release is published. `amux-bin` consumes the released
glibc tarballs from the GoReleaser pipeline; it does **not** build from source
and is **not** generated by GoReleaser (ADR-0007). Publishing to the AUR is a
separate, explicitly user-authorized operation — never done automatically.

Files: `packaging/aur/PKGBUILD`, `packaging/aur/.SRCINFO`,
`packaging/aur/amux.install`, `packaging/systemd/amuxd.user.service`.

## What the package guarantees

- Installs `amux` + `amuxd` to `/usr/bin`, completions for bash/zsh/fish, docs
  to `/usr/share/doc/amux/`, and — **disabled** — the example systemd *user*
  unit to `/usr/lib/systemd/user/amuxd.service`.
- **Never** enables or starts a service. `amux.install` only prints how the user
  starts it themselves (`systemctl --user enable --now amuxd`). Auto-starting an
  untrusted daemon on install is forbidden by the autonomy policy.
- `options=('!strip')` — binaries are already stripped (`-s -w`); keep the
  reproducible release bytes rather than re-stripping.
- `sha256sums_*` are **real digests, never `SKIP`**. `SKIP` on a binary package
  defeats supply-chain integrity (ADR-0007 hash-lock posture).

## Update procedure (per release)

1. **Bump the version.** In `PKGBUILD` set `pkgver=X.Y.Z` (no leading `v`) and
   reset `pkgrel=1`. If only packaging metadata changed (not the upstream
   version), keep `pkgver` and bump `pkgrel`.
2. **Set the release base URL.** `url=` → the canonical project URL;
   `_release_base` resolves to `${url}/releases/download/v${pkgver}`. Confirm the
   `source_x86_64` / `source_aarch64` URLs resolve to the published tarballs
   (`amux_<ver>_linux_amd64.tar.gz` / `..._arm64.tar.gz`).
3. **Fill the checksums from the release `checksums.txt`** (the same file
   verified per `artifact-verification.md`):

   ```bash
   grep linux_amd64 checksums.txt   # -> sha256sums_x86_64
   grep linux_arm64 checksums.txt   # -> sha256sums_aarch64
   ```

   Paste each digest; **never** `SKIP`.
4. **Set the license.** Replace `license=('custom')` with the SPDX id once a
   `LICENSE` file ships in the tarball.
5. **Regenerate `.SRCINFO`** (a stale one is an AUR-side defect):

   ```bash
   cd packaging/aur && makepkg --printsrcinfo > .SRCINFO
   ```

6. **Build & test in a clean chroot** (never test only on your dirty host):

   ```bash
   # from packaging/aur, on an Arch host with devtools installed
   extra-x86_64-build          # clean chroot build (x86_64)
   # aarch64: use an aarch64 Arch host / qemu-user chroot with the arm64 tarball
   namcap PKGBUILD amux-bin-*.pkg.tar.zst   # lint metadata + built package
   ```

   A clean chroot proves the package installs with no undeclared host state.
7. **Smoke the installed binaries** (distro-agnostic harness):

   ```bash
   packaging/smoke/smoke-install.sh amux_<ver>_linux_x86_64.tar.gz
   ```

   Asserts both binaries run, report a stamped `--version`, ship all three
   completion files, and are cgo-free — the linkage check is **fail-closed**: a
   dynamically linked / libc-dependent binary (a cgo regression) fails the smoke
   gate rather than passing with a note (see `packaging/smoke/lib-linkage.sh`).
8. **Publish** (user-authorized): commit `PKGBUILD` + `.SRCINFO` to the AUR git
   remote and push. Announce the version.

## Downgrade / yank

If a release is bad, users pin the prior version by installing the previous
tarball or the prior AUR commit; see `rollback-and-recovery.md` §"Artifact
rollback". Never rewrite a published AUR history to hide a bad build — supersede
it with a new `pkgrel`/`pkgver`.

## Deferrals (honest)

`makepkg`, `extra-x86_64-build`, `namcap`, and a clean chroot are **Arch-only**
and cannot run on macOS or a non-Arch CI host. On a dev host the PKGBUILD and
`.install` are validated by `bash -n` (syntax) and reviewed against this
procedure; the chroot build + `namcap` are executed by QA (Q8) / the release
owner on a real Arch host. The checked-in `pkgver`/`sha256sums` are placeholders
until a real release fills them.
